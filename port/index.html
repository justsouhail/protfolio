Simulating Secure Enterprise Connectivity

I had the opportunity to design a network architecture on Packet Tracer (both physical and logical modes) aimed at simulating real production scenarios. This architecture relied on several key components: dynamic and static routing, EtherChannel, web services, DHCP, DNS, inter-VLAN routing, VLANs, and an IPsec over GRE tunnel to ensure reliable and secure connectivity.

I started by creating a LAN in Casablanca using a 2-tier architecture. In this LAN, I configured four VLANs to separate services while ensuring redundancy through dual links and EtherChannel between the Layer 3 switches. I then replicated the same configuration in the Rabat LAN, and the connection between the two sites was established through an ISP network based on OSPF routing. To establish a point-to-point connection, a GRE tunnel was configured between the two sites. However, as this tunnel was not secure, an IPsec tunnel was implemented to encrypt the communications, forming an overlay network.

Next, to continue the simulation, the datacenter LAN was created with two web servers simulating (facebook.com and google.com), along with a DNS server.

Communication between the different LANs occurs through the ISP network, which ensures redundant links while relying on OSPF for routing.

VoIP--IP-Telephony-System

Designing and implementing a VoIP network for IP Telephony Systems

I am delighted to present to you a project I am currently undertaking. It involves designing and implementing a VoIP network for IP Telephony Systems. In this network, each desktop is equipped with its own telephone set, directly connected to the PC rather than through a switch. The infrastructure comprises four servers - DHCP, EMAIL, DNS, and HTTP - situated at the server-side site. Here are the key technologies utilized:

1. Creation of data and voice VLANs with port VLAN assignments.

2. Subnetting and IP Addressing

3. Configuration of VoIP or Telephony services on all routers.

4. Setting up routing for VoIP or Dial peering on all routers.

5. Inter-VLAN Routing configured on routers using the router-on-a-stick method.

6. Dedicated DHCP Server for data, providing dynamic IP allocation.

7. Routers serving as DHCP servers for Voice, allocating dynamic IPs to IP Phones.

8. SSH configured for secure remote access.

9. OSPF implemented as the routing protocol for optimal network routing.
